Chat Sending Functionality

This page demonstrates the new chat sending functionality implemented in the user input store.

Features:

  • Multiple Model Support: Select multiple models to send asynchronous requests
  • Single Model Support: Select one model for normal streaming response
  • Authentication: Uses user_id and token from global state
  • FormData Format: Sends data in the expected FormData format to /user/chat
  • Error Handling: Proper error handling and status reporting

Current Implementation:

  • Uses full_input_messages: [] (empty array as requested)
  • Converts model_ids to model names for backend
  • Includes extra_request_list from user input store
  • Handles both success and error responses
