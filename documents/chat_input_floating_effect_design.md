# Chat 输入栏悬浮视觉效果 设计文档

## 背景
现有输入栏虽然固定在页面底部，但背景色与聊天内容区域相近，阴影较弱，层次感不足，导致视觉上与聊天消息处于同一平面。新的设计需要让输入栏呈"悬浮"状态，明显高于聊天内容区域。

## 设计目标
1. 维持 `fixed` `bottom-0` 布局，保证输入栏始终固定在视口底部。
2. 通过增强阴影、增加顶角圆角、提高 `z-index` 等手段，让输入栏看起来浮起。
3. 在深/浅色模式下均保持良好对比度。
4. 不影响现有功能与交互体验。

## 设计要点
| 要点 | 说明 |
| ---- | ---- |
| **背景色** | 使用纯白 `bg-white`（浅色模式）+ 略带透明度的 `backdrop-blur`，与灰色聊天背景形成对比 |
| **阴影** | 自定义 `shadow-[0_-6px_16px_rgba(0,0,0,0.08)]`，向上投影，制造悬浮感 |
| **圆角** | 仅顶部使用 `rounded-t-2xl`，下边仍与屏幕底部贴合 |
| **边框** | `border-t-2 border-gray-200` 加粗顶部边界，进一步分层 |
| **层级** | 提升到 `z-50`，确保覆盖任何聊天内容及其他浮层 |
| **毛玻璃** | `backdrop-blur-sm` 轻微模糊后方内容，提升立体层次 |

## 技术实现
1. **修改 `ChatInput.tsx`**
   * 更新最外层 `div` 的 Tailwind 类名，加入以上样式。
   * 使用 React Fragment 包裹返回的 JSX，保证只有一个根节点。
2. **验证**
   * 检查在不同消息数量、滚动位置下，输入栏阴影是否可见。
   * 观察移动端横竖屏表现。

## 可选优化
* 如阴影在深色模式下过弱，可动态替换 `shadow` 颜色。
* 调整 `ChatMessageList` `pb-40` 余量，以适配输入栏真实高度。

## 风险评估
* 仅涉及样式与极小量 JSX 变动，无逻辑修改，风险极低。
* 若浏览器不支持 `backdrop-blur`，效果仍退化为普通白色背景，不影响使用。
