# 聊天界面三项 Bug 修复 – 设计文档

> 版本：v1.0 – 2025-07-03
> 负责人：AI

## 1. 问题概述
| 编号 | 描述                                                                                                                                       |
| ---- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| B-1  | Agent 思维路径（<think>…</think> 包裹，灰色文字）无法跟随"agent 思维路径"按钮正确隐藏/显示，导致关闭侧边面板后气泡内依旧展示灰色思考内容。 |
| B-2  | Iframe 预览默认 `transform: scale(0.65)` 导致网页整体缩小，阅读体验差。                                                                    |
| B-3  | 底部输入容器阴影仅向上投射，模型选择行下方缺乏阴影或轮廓，视觉割裂。                                                                       |

## 2. 解决方案一览
| 编号 | 方案要点                                                                                                                                                                                                                                                            |
| ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| B-1  | 1) 在 `ChatMessage.tsx` 的气泡最外层动态加入 `hide-thinking` class。<br/>2) 在 `index.css` 中添加 `.hide-thinking .thinking-content { display:none; }` 规则。<br/>3) 切换逻辑与侧边面板 `showThoughtPanel` 同步：面板关闭 → 隐藏思考文字；面板打开 → 展示思考文字。 |
| B-2  | 移除 `.safe-iframe-wrapper iframe` 上的 `transform: scale(0.65)` 与 `transform-origin`；并取消任何 `/proxy?url=` 前端改写，直接使用原始 URL 嵌入。                                                                                                                  |
| B-3  | 将输入容器 Tailwind 自定义阴影由单向改为双向：`shadow-[0_-6px_16px_rgba(0,0,0,0.08),0_6px_12px_rgba(0,0,0,0.04)]`，为底部增加 6px 轻阴影，视觉上形成完整卡片效果。                                                                                                  |

## 3. 详细设计
### 3.1 B-1 思维路径显示/隐藏
- **状态同步**：利用现有 `showThoughtPanel` 布尔状态；当其为 `false` 时给气泡额外 class `hide-thinking`。
- **样式隔离**：新增 CSS 选择器 `.hide-thinking .thinking-content { display:none; }`，无需重渲染 HTML，即可通过 class 切换完成隐藏。
- **交互流**：
  1. 用户点击 "agent思维路径" 按钮 → `showThoughtPanel` 取反。
  2. `ThoughtPanel` 组件根据该 state 决定显隐。
  3. 同时气泡 class 更新，灰色思考文字随之显/隐。

### 3.2 B-2 Iframe 缩放
- **问题来源**：早期为"缩略图"需求手动缩放至 65%，但在真实使用中几乎所有资源需要正常比例展示。
- **改动**：删除缩放与 origin 设置，保留宽高 100%。
- **兼容**：全屏模式 CSS (`iframe:fullscreen`) 仍显式 `transform:none`，确保无锁定缩放。

### 3.3 B-3 输入容器阴影
- **视觉需求**：提供顶部向上浮起效果的同时，需要轻微底部阴影避免内容延伸至模型选择区的突兀分割。
- **实现**：使用 Tailwind Arbitrary Value 组合双 box-shadow。
  ```html
  shadow-[0_-6px_16px_rgba(0,0,0,0.08),0_6px_12px_rgba(0,0,0,0.04)]
  ```

## 4. 影响范围
- `ChatMessage.tsx`、`index.css`、`ChatInput.tsx` 三处前端文件；无后端改动。
- 无破坏性数据库/接口变动。

## 5. 回归验证
1. **思维路径**：
   - 触发带 `<think>` 内容的回复。
   - 点击 "agent思维路径" 按钮：侧边面板打开且气泡灰色文字可见。
   - 再次点击或在面板右上角 × 关闭：面板消失且气泡灰色文字隐藏。
2. **Iframe**：发送 `<https://example.com>` 链接，检查 iframe 以 100% 尺寸渲染，无缩放。
3. **输入容器**：在浅色背景下观察模型选择行下方阴影，视觉连贯。

## 6. 风险评估
- Tailwind Arbitrary Shadow 在旧版 Safari 可能不支持多 shadow，但影响极小。
- 思维路径隐藏依赖于单条气泡的 class，不影响其他组件。
