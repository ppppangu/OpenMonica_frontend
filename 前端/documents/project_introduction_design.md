# 通用 AI Agent 平台介绍文档

## 1. 项目定位与核心价值

本平台定位为"**通用 AI Agent 平台**"，致力于帮助企业或个人以最低的成本快速构建、部署并运营具备知识增强能力的智能助手。它强调 **交互优先** 和 **功能可扩展** 两大核心理念：

* **交互优先**：从 UI/UX 设计到系统 API，全链路围绕"对话即入口"展开，通过流式输出、富文本及多模态支持，为终端用户提供媲美本地 App 的顺滑体验。
* **功能可扩展**：采用高度模块化的技术架构，以"知识库 + 工具链 + 多模态"三大插件层级为核心，支持按需组合与热插拔，满足持续演进的业务需求。

凭借上述设计，平台可以在多行业场景下快速落地，帮助用户将各类私有知识资产转化为可交互、可迭代的智能生产力。

## 2. 核心特点

**1. 多模态交互能力**

**输入多模态**

* 文本：实时对话
* 图片：支持直接上传图片作为
* 其他格式文件：.py,.rs,.md等，agent使用工具提取其他各类通用格式的信息

**输出多模态**

分为**文本**和其他各类的**多模态信息**

* 文本：流式输出 + 高亮代码块 + mermaid图表支持
* artifact：制作的图表(思维导图，流程图，关系图)，多模态作品实时展示和下载（简历，网页，交互式ppt ）

## 3. 与传统 RAG 系统的差异化特点

| 维度       | 传统 RAG 局限                                                | 本平台突破                                                                                                                                     |
| ---------- | ------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| 交互方式   | 以文本接口为主，缺少多模态输入和输出体验；                   | 实现完整的**富文本 + 流式渲染**对话；支持拖拽上传全格式文件、粘贴图片、嵌入网址等多模态输入，以及                                        |
| 功能扩展性 | 多数框架以"检索→拼接→生成"单一流程为核心，难以加载外部工具 | **统一工具调用层**，实现"函数调用"式插件扩展，可轻松串联搜索、翻译、数据查询等外部能力                                                   |
| 用户体验   | 缺少知识库可视化，管理成本高                                 | V2 版本 (`components/knowledgebase_v2`) 内置 **知识图谱视图（`KBGraphView.tsx`）** 与批量导入/同步能力，让非技术用户也能直观管理数据 |
| 可复用性   | 与业务强耦合，二次开发门槛高                                 | 架构清晰，**UI、State、API、Tool 、三层解耦**；任何功能都可以在不改动核心代码的情况下以"插件"形式增加或替换                              |

## 4. 技术架构亮点

1. **流式响应与实时交互**

   * 利用原生 `Fetch + ReadableStream`（见 `utils/streamingUtils.ts`）与 Vite 热更新，实现 <200ms 首字节
   * 渲染层采用 **逐 token 渲染**，配合虚拟滚动保持高性能
2. **模块化组件设计**

   * 组件集中于 `src/components`，命名遵循领域优先原则（如 `chat/`, `file/`, `knowledgebase_v2/`），方便按业务拼装
3. **多文件类型支持**

   * 通过 `utils/fileManagementApi.ts` 与 `components/file/FileUpload.tsx` ，覆盖 Office、Markdown、图片、音频等主流格式，自动展示预览或解析结果
4. **可视化知识图谱**

   * `KBGraphView.tsx` 借助 Mermaid 与 Force-Graph 渲染知识节点及其引用关系，可直接在前端编辑关系或合并节点

| 法律/医疗等高合规行业 | 私有文档问答、流程合规检查           | 前端独立部署，后端可对接私有大模型，保证数据安全     |
